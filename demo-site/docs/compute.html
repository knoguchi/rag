Demo Compute

Demo Compute provides scalable virtual machines and container services for running your applications in the cloud.

Instance Types

Choose from a variety of instance types optimized for different workloads:

Balanced compute, memory, and networking for general-purpose workloads.

Creating an Instance

You can create instances via the CLI, API, or web console.

Using the CLI

# Create a standard instance
demo compute create \
  --name web-server \
  --type standard-2 \
  --image ubuntu-22.04 \
  --region us-east-1

# Create with SSH key
demo compute create \
  --name web-server \
  --type standard-2 \
  --image ubuntu-22.04 \
  --ssh-key ~/.ssh/id_rsa.pub

Using the API

curl -X POST https://api.demo.cloud/v1/instances \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "web-server",
    "type": "standard-2",
    "image": "ubuntu-22.04",
    "region": "us-east-1"
  }'

Available Images

Demo Compute supports a variety of operating system images:

  • Ubuntu: ubuntu-22.04, ubuntu-20.04, ubuntu-18.04
  • Debian: debian-12, debian-11
  • CentOS: centos-stream-9, centos-stream-8
  • Fedora: fedora-39, fedora-38
  • Alpine: alpine-3.19, alpine-3.18
  • Windows Server: windows-2022, windows-2019

Networking

Each instance gets:

  • A private IP address within your VPC
  • Optional public IP address (IPv4 and/or IPv6)
  • Up to 10 Gbps network bandwidth

Security Groups

Control inbound and outbound traffic with security groups:

demo security-group create web-sg \
  --inbound "tcp:80:0.0.0.0/0" \
  --inbound "tcp:443:0.0.0.0/0" \
  --inbound "tcp:22:10.0.0.0/8"

demo compute update web-server \
  --security-groups web-sg

Auto-Scaling

Automatically scale your instances based on CPU, memory, or custom metrics:

demo autoscale create web-scale \
  --min 2 \
  --max 10 \
  --metric cpu \
  --target 70 \
  --instance-type standard-2 \
  --image ubuntu-22.04

Snapshots and Backups

Create point-in-time snapshots of your instances:

# Create a snapshot
demo compute snapshot create \
  --instance web-server \
  --name "before-upgrade"

# Restore from snapshot
demo compute create \
  --name web-server-restored \
  --snapshot snap_abc123
