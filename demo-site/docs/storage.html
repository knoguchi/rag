Demo Storage

Demo Storage is an S3-compatible object storage service with 99.999999999% (11 nines) durability. Store and retrieve any amount of data at any time.

Key Features

  • Highly Durable: Data is automatically replicated across multiple facilities
  • Scalable: Store unlimited objects with no size limits
  • S3 Compatible: Use existing S3 tools and SDKs
  • Secure: Server-side encryption with your keys or ours
  • Cost-Effective: Pay only for what you use

Storage Classes

Class Use Case Availability Price/GB/month
Standard Frequently accessed data 99.99% $0.023
Infrequent Access Monthly access patterns 99.9% $0.0125
Archive Long-term storage 99.9% $0.004

Creating a Bucket

Buckets are containers for your objects. Each bucket name must be globally unique.

Using the CLI

# Create a bucket
demo storage bucket create my-bucket --region us-east-1

# List buckets
demo storage bucket list

# Delete a bucket (must be empty)
demo storage bucket delete my-bucket

Using the API

curl -X POST https://api.demo.cloud/v1/storage/buckets \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-bucket",
    "region": "us-east-1"
  }'

Working with Objects

Upload Files

# Upload a single file
demo storage upload my-bucket/images/logo.png ./logo.png

# Upload a directory
demo storage upload my-bucket/assets/ ./assets/ --recursive

# Upload with metadata
demo storage upload my-bucket/report.pdf ./report.pdf \
  --metadata "author=John,department=Sales"

Download Files

# Download a file
demo storage download my-bucket/images/logo.png ./logo.png

# Download a directory
demo storage download my-bucket/assets/ ./assets/ --recursive

List Objects

# List all objects
demo storage ls my-bucket

# List with prefix
demo storage ls my-bucket/images/

# List with details
demo storage ls my-bucket --long

Access Control

Control who can access your buckets and objects with bucket policies and ACLs.

Make a Bucket Public

demo storage bucket policy my-bucket --public-read

Generate Presigned URLs

Share private objects temporarily with presigned URLs:

# Generate a download URL valid for 1 hour
demo storage presign my-bucket/report.pdf --expires 1h

# Generate an upload URL
demo storage presign my-bucket/uploads/new-file.pdf \
  --method PUT --expires 30m

S3 Compatibility

Demo Storage is fully compatible with the S3 API. Use your favorite S3 tools:

Using AWS CLI

aws configure set aws_access_key_id YOUR_ACCESS_KEY
aws configure set aws_secret_access_key YOUR_SECRET_KEY

aws s3 --endpoint-url https://storage.demo.cloud ls s3://my-bucket/
aws s3 --endpoint-url https://storage.demo.cloud cp ./file.txt s3://my-bucket/

Using boto3 (Python)

import boto3

s3 = boto3.client(
    's3',
    endpoint_url='https://storage.demo.cloud',
    aws_access_key_id='YOUR_ACCESS_KEY',
    aws_secret_access_key='YOUR_SECRET_KEY'
)

# Upload a file
s3.upload_file('local-file.txt', 'my-bucket', 'remote-file.txt')

# List objects
response = s3.list_objects_v2(Bucket='my-bucket')
for obj in response.get('Contents', []):
    print(obj['Key'])

Lifecycle Policies

Automatically transition or delete objects based on age:

demo storage lifecycle my-bucket \
  --transition-to-ia-days 30 \
  --transition-to-archive-days 90 \
  --expire-days 365

Versioning

Keep multiple versions of objects to protect against accidental deletion:

# Enable versioning
demo storage bucket versioning my-bucket --enable

# List versions
demo storage ls my-bucket/important-file.txt --versions

# Restore a previous version
demo storage restore my-bucket/important-file.txt --version v123

Encryption

All data is encrypted at rest by default. You can also use your own keys:

# Upload with custom encryption key
demo storage upload my-bucket/secret.txt ./secret.txt \
  --sse-customer-key YOUR_BASE64_KEY
